# Правило: добавление музыкального исполнителя в `artists.csv`

Как агенту находить данные и корректно дописывать артиста в базу игры «Ноталь». База — плоский CSV в корне репозитория (`artists.csv`), отсортированный по популярности **по убыванию**.

---

## 0. Прежде чем начать

1. **Проверь, что артиста ещё нет** в `artists.csv` (по колонке `name`, с учётом регистра и латиницы/кириллицы).
2. При любом **реальном разночтении спрашивай пользователя**, не выбирай молча (типовые развилки — в конце файла).
3. Правило приоритета качества: **лучше оставить поле пустым/спорное вынести на вопрос, чем записать неверные данные.** Особенно это касается фото (см. §5).

---

## 1. Формат CSV

Заголовок (порядок колонок можно менять — сопоставление идёт по имени колонки):

```
name,debut_year,members,gender,genre,city,region,popularity,image,track_title,track_link,track_cover
```

- Разделитель — запятая. **Поле, содержащее запятую, — в двойных кавычках** (`"Ленинский (Экибастуз), Павлодарская область"`).
- Кодировка UTF-8. Пустое необязательное поле — просто ничего между запятыми.
- Одна строка = один артист.

---

## 2. Поля: что писать и где искать

| Поле | Что писать | Источник |
|---|---|---|
| `name` | Имя/название как принято на сцене (по нему идёт поиск и проверка ответа). | — |
| `debut_year` | Целое. Для **групп** — год основания (в базе так: Машина времени 1969, КИНО 1982, Nautilus Pompilius 1983). Для **сольных** — год начала карьеры / первого релиза. | ru.wikipedia |
| `members` | Число участников самого известного состава. **`1` = соло.** Сольный проект-персонаж (напр. один автор под псевдонимом) = `1`, даже если на сцене выступает бэнд. | ru.wikipedia |
| `gender` | Ровно одно: `Мужской` / `Женский` / `Смешанный` / `Небинарный`. | — |
| `genre` | Ровно **один** из закрытого списка (см. §3). | ru.wikipedia / здравый смысл |
| `city` | Город происхождения (точное совпадение → зелёная плитка). Историческое имя допустимо в скобках: `Свердловск (ныне Екатеринбург)`. | ru.wikipedia |
| `region` | Для РФ — **федеральный округ** (см. §3). Для зарубежья — **название страны** (`Украина`, `Беларусь`, `Казахстан`, `Азербайджан` …). | ru.wikipedia |
| `popularity` | Целое = **число слушателей в месяц на Яндекс.Музыке**. Больше = популярнее; игра сама строит рейтинг «№место». | music.yandex.ru (страница артиста) |
| `image` | Прямой хотлинк на фото с `upload.wikimedia.org` (или пусто). См. §5. | Wikipedia pageimages API |
| `track_title` | Один характерный/узнаваемый трек. | music.yandex.ru |
| `track_link` | Ссылка на прослушивание. Прямой трек предпочтителен: `https://music.yandex.ru/album/<A>/track/<T>`. Если прямой не достаётся — поисковая: `https://music.yandex.ru/search?text=<Имя>%20<Трек>` (пробел → `%20`, кириллицу можно оставлять как есть). | music.yandex.ru |
| `track_cover` | Необязательно. URL обложки трека или пусто. | — |

## 3. Закрытые списки

**Жанры (ровно один):** Поп · Рок · Рэп/Хип-хоп · Шансон · Электроника · Панк · Инди · Метал · Фолк · Авторская песня · Эстрада · R&B · Джаз.
Список задаётся в `index.html` (константа `GENRES`). Если артист явно гибрид (напр. фолк-драм-н-бейс) — **спроси пользователя**, какой жанр ставить.

**Федеральные округа РФ (для `region`):** Центральный · Северо-Западный · Южный · Северо-Кавказский · Приволжский · Уральский · Сибирский · Дальневосточный.
Не из РФ → пиши страну.

---

## 4. Популярность и позиция в файле

1. Открой страницу артиста на `music.yandex.ru`, возьми число из строки «N слушателей в месяц» → это `popularity` (целое, без пробелов).
2. Число «живое» (меняется) — это нормально, фиксируем срез на момент добавления.
3. **Вставляй строку на правильную позицию** так, чтобы файл остался отсортирован по `popularity` по убыванию. Найди соседей по величине и вставь между ними.
4. Если у артиста несколько профилей на Яндексе (напр. соло vs группа) — бери профиль, соответствующий тому, как назван артист в `name`; при большом расхождении **спроси пользователя**.

## 5. Картинка (`image`) — самое хитрое

### Почему хотлинк с Википедии работает
- Игра просто вставляет `<img src=…>` с фолбэком на кружок с буквой (`index.html`, `avatarEl`). **CORS не нужен** (canvas не читается).
- Wikimedia отдаёт файлы на сторонние сайты: **Referer не блокируется**, заголовок `access-control-allow-origin: *`.
- Вёрстка не ломается от большого файла: аватар — фикс. бокс (34/56px) с `overflow:hidden` + `object-fit:cover`. Оригинал влияет только на трафик.

### Обязательные правила
1. **URL превью бери ТОЛЬКО из API.** Wikimedia не генерирует произвольные размеры — самодельный `…/440px-File.jpg` даёт `HTTP 400`. Используй `pithumbsize=320`; если thumbnail нет — бери `original`.
2. **Найди ПРАВИЛЬНУЮ статью** именно про этого артиста. Плоское имя часто ведёт на статью не по теме, у которой ТОЖЕ есть картинка (ловушки: Каста→иллюстрация «касты», Машина времени→DeLorean, Мельница→ветряк, Пурген→химия, Дельфин→дельфин). Типовые заголовки:
   - группы: `Название (группа)` — Ария→`Ария (группа)`, КИНО→`Кино (группа)`, Каста→`Каста (группа)`;
   - рэп: `Имя (рэпер)` / `(музыкант)` — Баста→`Баста (музыкант)`, MACAN→`Macan`, GUF→`Guf`;
   - сольные: `Фамилия, Имя Отчество`.
   Сверяй интро статьи (город/жанр) — тот ли это артист.
3. **Смысловая проверка важнее покрытия.** Картинка должна быть фото артиста/группы либо их логотипом. Химия/ветряк/машина/тюрьма/ноты/карта — **отбросить**. Нет подходящей → оставить пусто (игра покажет букву). **Чужое лицо недопустимо.**
4. Предпочитай `commons` локальным несвободным файлам (`/wikipedia/ru/…`) — они стабильнее. Несвободные тоже хотлинкаются; их прямой URL достаётся через `imageinfo`.
5. **Проверь ссылку:** должно быть `200` и `image/*`. `429` = троттлинг Wikimedia (частые запросы с одного IP), ссылка при этом валидна — принимай её.

### Рецепты (curl)
Поиск правильной статьи-кандидата:
```bash
curl -s -G "https://ru.wikipedia.org/w/api.php" \
  -H "User-Agent: rus-spotle/1.0 (https://cash-metall.github.io/rus-spotle/)" \
  --data-urlencode action=query --data-urlencode format=json \
  --data-urlencode list=search --data-urlencode "srsearch=<ИМЯ>" --data-urlencode srlimit=6
```
Фото 320px + интро статьи (для сверки, что артист тот):
```bash
curl -s -G "https://ru.wikipedia.org/w/api.php" \
  -H "User-Agent: rus-spotle/1.0 (https://cash-metall.github.io/rus-spotle/)" \
  --data-urlencode action=query --data-urlencode format=json \
  --data-urlencode prop="pageimages|extracts" \
  --data-urlencode piprop="thumbnail|original|name" --data-urlencode pithumbsize=320 \
  --data-urlencode exintro=1 --data-urlencode explaintext=1 --data-urlencode exchars=180 \
  --data-urlencode redirects=1 --data-urlencode "titles=<ТОЧНЫЙ_ЗАГОЛОВОК>"
```
Возьми `thumbnail.source` (иначе `original.source`). Проверка ссылки:
```bash
curl -s -o /dev/null -w "%{http_code} %{content_type}\n" \
  -H "User-Agent: Mozilla/5.0 ... Chrome/120 Safari/537.36" \
  -H "Referer: https://cash-metall.github.io/rus-spotle/index.html" "<URL>"
```

> Кириллица в URL из API уже percent-encoded (`%D0%9A…`) — при программной проверке дополнительно квотируй путь, иначе HTTP-клиент упадёт на не-ASCII.

## 6. Массовое добавление (много артистов сразу)

- Подбор фото плохо детерминируется (неоднозначные заголовки) — **используй агентов** с судейской проверкой статьи, а не «тупой» скрипт по имени.
- Дели список на чанки (~15 артистов), по одному агенту на чанк; каждый пишет результат в свой файл (`name<TAB>url<TAB>title<TAB>note`), потом слияние по **точному** совпадению имени.
- Факты/популярность/трек можно параллелить так же (один агент на артиста/чанк).

## 7. Чеклист перед сохранением

- [ ] Артиста ещё не было в базе.
- [ ] Ровно 12 полей; поля с запятыми — в кавычках.
- [ ] `genre` из закрытого списка (§3); `gender` — из четырёх значений; `region` — округ РФ или страна.
- [ ] `popularity` — целое; строка вставлена так, что файл остался отсортирован по убыванию.
- [ ] `image` — пусто или `https://upload.wikimedia.org/…`, ссылка проверена (`200 image/*` или `429`), фото соответствует артисту.
- [ ] `track_link` открывается / ведёт на нужный трек.

Валидация (PowerShell):
```powershell
$r = Import-Csv artists.csv -Encoding UTF8
$g = @('Поп','Рок','Рэп/Хип-хоп','Шансон','Электроника','Панк','Инди','Метал','Фолк','Авторская песня','Эстрада','R&B','Джаз')
"строк: $($r.Count)"
"жанры вне списка: $(($r | ? { $g -notcontains $_.genre }).Count)"
$p = $r | % { [int64]$_.popularity }; $d=$true; for($i=1;$i -lt $p.Count;$i++){ if($p[$i] -gt $p[$i-1]){$d=$false} }
"по убыванию: $d"
"image не с wikimedia: $(($r | ? { $_.image -and $_.image -notmatch '^https://upload\.wikimedia\.org/' }).Count)"
```

## 8. Когда спрашивать пользователя (типовые развилки)

- **Жанр-гибрид** — что выбрать из закрытого списка (напр. Электроника vs Фолк).
- **Год дебюта** — год основания vs год первого альбома; для сольных — начало карьеры vs первый релиз.
- **Состав соло-персонажа** — `1` (автор) vs число участников сценического бэнда.
- **Популярность** — если у артиста несколько профилей на Яндексе или сильный разброс данных (раздробленный каталог старых групп).
- **Место происхождения** — если рождение и «вырос/сформировался» в разных местах/странах.
