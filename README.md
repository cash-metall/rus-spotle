# Ноталь — русский аналог Spotle

Ежедневная угадайка про русских музыкантов в духе Wordle/Spotle. Без зависимостей.
Игра (`index.html`) подгружает базу артистов (`artists.csv`) по сети с локального сервера.

## Файлы
- **`index.html`** — сама игра. Базу не хранит — запрашивает её через `fetch('artists.csv')`.
- **`artists.csv`** — рабочая база (171 артист, 13 жанров). Именно её отдаёт сервер.
- **`artists-template.csv`** — чистый шаблон с примерами: образец формата.
- **`icon.svg`** — иконка приложения (микрофон), используется как favicon и в шапке.

## Как запустить
Игру нужно открывать **через локальный веб-сервер** (иначе браузер заблокирует `fetch`
по протоколу `file://`). Сервер уже поднят на порту **8081** в этой папке:

- Откройте в браузере **http://localhost:8081/** — игра сама подтянет `artists.csv`.

Если сервер не запущен, поднять его можно из папки игры, напр.:
```
python -m http.server 8081
```
Адрес базы задаётся константой `DB_URL` в начале скрипта `index.html` (по умолчанию
относительный `artists.csv`, т.е. тот же origin, что и страница).

## Как обновить базу
- **Правкой файла:** отредактируйте `artists.csv` (тот же формат, что и шаблон) и обновите
  страницу — сервер отдаст новую версию.
- **Через игру (локально, поверх сервера):** Меню (☰) → «Загрузить свой CSV» → выберите файл.
  Он сохранится в браузере (localStorage) и получит приоритет над серверной базой.
  Вернуться к серверной — Меню → «Перезагрузить базу с сервера».

## Формат CSV
Первая строка — заголовок, порядок колонок можно менять (сопоставление по названию):

| колонка | что писать |
|---|---|
| `name` | Имя артиста/группы (по нему идёт поиск и проверка ответа) |
| `debut_year` | Год **первого релиза** (альбом/EP/микстейп), целое число |
| `members` | Число участников самого известного состава; `1` = «Соло» |
| `gender` | `Мужской` / `Женский` / `Смешанный` / `Небинарный` |
| `genre` | Ровно один жанр из списка (см. ниже) |
| `city` | Город происхождения (точное совпадение → зелёный) |
| `region` | Регион/округ или страна для «близко» (тот же регион → жёлтый) |
| `popularity` | Любое число, где **больше = популярнее** (напр. число слушателей). Игра сама построит рейтинг «№место» |
| `image` | (необязательно) URL/дата-URI аватара. Пусто → кружок с буквой |
| `track_title` | Один характерный трек для финального экрана |
| `track_link` | Ссылка на прослушивание (Яндекс.Музыка / VK / Zvuk / Apple) |
| `track_cover` | (необязательно) URL обложки трека |

**Жанры (закрытый список):** Поп, Рок, Рэп/Хип-хоп, Шансон, Электроника, Панк, Инди,
Метал, Фолк, Авторская песня, Эстрада, R&B, Джаз. Список задаётся в `index.html`
(константа `GENRES`) — при желании поправьте под свою сцену.

Поля с запятой внутри берите в двойные кавычки (`"..."`) — стандартный CSV.

## Механика (как в оригинале)
- 8 попыток на артиста дня (выбирается детерминированно от даты).
- 6 плиток: **Дебют, Состав, Популярность, Пол, Жанр, Город**.
- 🟩 точно · 🟨 близко · ⬛ мимо. Пороги «близко»: ±5 лет (дебют), ±~10% базы
  (популярность), тот же регион (город). У числовых плиток стрелка ▲/▼ показывает
  направление к настоящему значению.
- Статистика (сыграно/винрейт/серии/распределение) и «Поделиться» (эмодзи-сетка) —
  всё в localStorage.

## Настройки в коде (начало `<script>` в `index.html`)
- `MAX_GUESSES` — число попыток (по умолчанию 8).
- `DEBUT_CLOSE` — порог «близко» по году (5).
- `GENRES` — список жанров.
- Режим «Практика» (случайный артист) и «Для дальтоников» — в меню игры.
