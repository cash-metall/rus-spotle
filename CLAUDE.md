# CLAUDE.md — Ноталь (rus-spotle)

Ежедневная угадайка про русских музыкантов в духе Wordle/Spotle. **Полностью нейрогенерированный** проект без зависимостей.
Онлайн: https://cash-metall.github.io/rus-spotle/index.html

## Структура

Плоский репозиторий, всё в корне:

| Файл | Что это |
|---|---|
| `index.html` | Вся игра в одном файле (HTML+CSS+JS, без зависимостей и сборки). База грузится через `fetch('artists.csv')`. |
| `artists.csv` | База артистов. Отсортирована по `popularity` **по убыванию**. Формат — см. `README.md` и правило ниже. |
| `icon.svg` | Иконка (микрофон), favicon и логотип. |
| `README.md` | Описание для GitHub. |

## Запуск

Статический сайт. Локально нужен любой веб-сервер (иначе `fetch` заблокируется на `file://`), напр. `python -m http.server`. В проде — GitHub Pages по ссылке выше.

## Ключевые места в `index.html`

- Константы в начале `<script>`: `MAX_GUESSES` (8), `DEBUT_CLOSE` (5), `GENRES` (13 жанров), `DB_URL` (`artists.csv`).
- `parseCSV` / `buildDataset` — разбор базы (нормализует CRLF, тримит поля, поддерживает кавычки).
- `avatarEl` — аватар артиста: `<img src=image>` с фолбэком на кружок с буквой.

## Правила

- **Добавление музыкального исполнителя в `artists.csv`** → читай [`.claude/rules/add-music-artist.md`](.claude/rules/add-music-artist.md): какие поля, где искать данные (Википедия — факты; Яндекс.Музыка — популярность и трек; Wikipedia API — фото), как избегать ловушек с картинками и как корректно вставлять строку.

## Договорённости

- `popularity` = число слушателей в месяц на Яндекс.Музыке; при добавлении артиста сохраняй сортировку файла по убыванию.
- Картинки артистов — прямые хотлинки на `upload.wikimedia.org` (не скачиваем; CORS не нужен). Размер превью бери только из API.
- `genre` — ровно один из закрытого списка `GENRES`; `region` — федеральный округ РФ или название страны.
- При реальных разночтениях в данных артиста — спрашивай пользователя, не выбирай молча.
